# 需求:更輕量的原生殼(/user)

## 情境
我有一個桌面應用 `nativeApp`(CIM Hybrid Edge Platform)。它現在用 **Electron** 當外殼。
我覺得 Electron「太大了」——每個 app 都自帶一整套 Chromium,打包肥、記憶體吃得多。

## 痛點
- 打包體積大(Electron runtime 本身就上百 MB)。
- 我聽說 **Tauri** 比較輕(用系統內建的 WebView,不夾帶整個瀏覽器)。
- **但我不確定直接換會不會出問題**——這個 app 不只是個網頁殼,它還要拉起 Python engine、開外部標註工具、管子程序。換殼會不會把這些搞壞?

## JTBD(我真正想完成的)
> 當我想把 app 變輕、又怕換框架踩雷時,
> 我想要**先有一個能真的跑起來的 Tauri 版本**,
> 好讓我**親眼確認它跟 Electron 版行為一樣**,並知道「換過去」要付出什麼代價、有哪些風險,
> 再決定要不要正式遷移。

## 成功的樣子(我能觀察到的)
- 一個在 `nativeApp_Light` 的版本,我能啟動它。
- 它開出視窗、載入我熟悉的 portal 畫面。
- 它能像現在一樣把 Python engine 拉起來、工具清單出得來、至少一個工具能跑。
- 有人告訴我:換 Tauri **哪裡沒問題、哪裡有風險、要改什麼**。

## 不在我關心範圍內
- 我不要求一次把「打包成可攜 exe / 簽章 / 自動更新」全部做完——先證明「能跑、行為對」。
- 我不要重寫 portal 或 Python engine。
